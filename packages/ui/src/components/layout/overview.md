# Overview

`packages/ui/src/components/layout`

## Purpose

Reusable page geometry and scroll ownership. These compositions prevent feature code from manually summing sticky heights,
inventing viewport calculations, or implementing inconsistent master/detail behavior.

## Areas

- page intros, section headers, content sections, and toolbars;
- master/detail and split layouts;
- scrollable panes and sticky route frames;
- measured sticky stacks and viewport-available-height propagation.

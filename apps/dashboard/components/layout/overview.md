# Overview

`apps/dashboard/components/layout`

## Purpose

Product-shell styling shared across feature routes. Generic layout mechanics belong in `@multi-coc/ui`; this folder owns
only Dashboard-specific shell and History surface rules.

## Key files

- `shell.css` — application header, navigation, Quick Paste trigger, and responsive shell composition.
- `history.css` — History page geometry and result presentation shared by History components.
